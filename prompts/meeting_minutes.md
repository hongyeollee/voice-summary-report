# 회의록 생성 프롬프트

'배경'을 고려하고, 첨부한 회의 녹취록을 바탕으로 핵심적인 회의록을 작성할 것.
가급적 회의록에서 언급된 내용은 누락하지 말 것.

## 출력 형식

아래 4개 섹션으로 구성된 회의록을 작성하고, JSON 형식으로 반환할 것.

### 1. 회의 개요
- 일시 (언급된 경우, 없으면 오늘 날짜)
- 목적 (회의의 주제/목적을 1~2문장으로)
- 참석자 (내용에서 언급된 경우)

### 2. 주요 논의 사항
- 핵심 내용 3~5가지 요약
- 각 항목은 충분한 맥락이 포함되도록 작성

### 3. 결정 사항
- 의사결정이 완료된 내용
- 구체적인 결정 내용과 그 이유를 포함

### 4. Action Items (향후 계획)
- 담당자 및 기한이 포함된 할 일 목록
- 형식: [담당자] 할 일 내용 (기한: YYYY-MM-DD 또는 "미정")

## JSON 출력 형식

```json
{
  "background": "이 회의를 하게 된 배경 설명 (음성 내용에서 추론)",
  "overview": {
    "date": "회의 일시",
    "purpose": "회의 목적",
    "attendees": ["참석자1", "참석자2"]
  },
  "key_discussions": [
    "주요 논의사항 1 (충분한 맥락 포함)",
    "주요 논의사항 2",
    "주요 논의사항 3"
  ],
  "decisions": [
    "결정사항 1 (이유 포함)",
    "결정사항 2"
  ],
  "action_items": [
    {"assignee": "담당자", "task": "할 일 내용", "deadline": "기한 또는 미정"},
    {"assignee": "담당자", "task": "할 일 내용", "deadline": "기한 또는 미정"}
  ]
}
```

## 규칙
- 한국어로 작성
- 회의록에서 언급된 내용은 가급적 누락하지 말 것
- 참석자가 명시되지 않으면 "명시되지 않음"
- 회의가 아닌 내용(강의, 독백 등)도 동일 형식으로 구조화
- JSON만 반환, 추가 설명 없이
